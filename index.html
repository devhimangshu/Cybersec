Module 1: Getting Started, Background

1.1 Installing R on Windows

Concept: R is a free, open-source programming language and environment. RStudio is an Integrated Development Environment (IDE) that makes working with R much easier.

  • Download R from CRAN (Comprehensive R Archive Network): cran.r-project.org .
  • Download RStudio Desktop (Free version) from rstudio.com .
  • Install R first, then RStudio.
  • RStudio provides a console, script editor, environment pane, plot pane, etc. for a better workflow.

Example (Conceptual):

You'd typically double-click the downloaded .exe files and follow the installation wizard.

1.2 Writing Code / Setting Your Working Directory (Windows)

Concept: R code can be written directly in the R Console (for single lines or testing). For multiple lines or saving your work, use an R Script file (.R extension) in RStudio's editor. The Working Directory is the default location where R reads files from and writes files to. It's crucial for managing your data and scripts.

  • In RStudio: Use File → New File → R Script to open a script editor.
  • Run code from a script: Select lines and press Ctrl+Enter (Windows) or Cmd+Enter (Mac).
  • Setting Working Directory:
    • Manually: Session → Set Working Directory → Choose Directory... (RStudio).
    • Programmatically: setwd("path/to/your/folder") . Use forward slashes / even on Windows, or double backslashes \\ .
  • Getting Working Directory: getwd()

Examples: 

# Writing code in a script and running
# This is a comment
result <- 5 + 3
print(result)

# Setting working directory (Windows example)
setwd("C:/Users/YourName/Documents/R_Projects")

# Getting current working directory
current_dir <- getwd()
print(current_dir)

1.3 Overview and History of R

Concept: R evolved from the S language, developed by Ross Ihaka and Robert Gentleman at the University of Auckland. It's now maintained by the R Core Team.

  • Powerful for statistical modeling, data analysis, graphics, and machine learning.
  • Open-source and highly extensible through packages (libraries of functions and data).
  • Vibrant community and extensive documentation.
  • Emphasizes vectorized operations for efficiency.

1.4 R Console Input and Evaluation

Concept: The R Console is an interactive command-line interface where you type R expressions, and R immediately evaluates them and prints the result.

  • Expressions are evaluated from left to right.
  • Assignment operator: <- (preferred) or =
  • Case-sensitive (e.g., myVar is different from myvar ).
  • Comments start with # .

Examples: 

# Simple expression evaluation
10 / 2 # Output: 5

# Variable assignment
x <- 10
y = 20 # less preferred
z <- x + y
print(z) # Output: 30

# Using functions
sqrt(25) # Output: 5

Module 2: Data Types - R Objects and Attributes

2.1 R Objects and Attributes

Concept: In R, everything is an object. Objects have attributes, which are metadata describing the object (e.g., names, dimensions, class).

  • class() : Returns the class of an object (its type).
  • typeof() : Returns the type of storage of an object.
  • attributes() : Returns all attributes of an object.

Examples: 

x <- c(1, 2, 3)
class(x) # "numeric"
typeof(x) # "double" (default numeric is double precision)
attributes(x) # NULL (atomic vectors don't have many attributes by default)

m <- matrix(1:4, nrow=2)
attributes(m) # Output includes 'dim' (dimensions)

2.2 Vectors and Lists

Concept: Atomic Vectors are homogeneous collections of elements of the same type. Lists are heterogeneous collections that can contain different types of elements.

  • Atomic Vectors types: logical, integer, numeric, complex, character, raw.
  • Lists can contain vectors, other lists, data frames, functions, etc.
  • Coercion hierarchy: logical → integer → numeric → character.
  • Accessing elements: [ ] for vectors, [ ] and [[ ]] for lists.

Examples: 

# Atomic Vectors
log_vec <- c(TRUE, FALSE, TRUE)
int_vec <- c(1L, 2L, 3L)
num_vec <- c(1.5, 2.0, 3.5)
char_vec <- c("apple", "banana", "cherry")

# Coercion example
mixed_vec <- c(1, "hello", TRUE)
class(mixed_vec) # "character" (all coerced to character)

# Lists
my_nested_list <- list(
  num_data = c(10, 20, 30),
  text_data = "sample text",
  is_active = TRUE,
  another_list = list(sub_item = 5)
)

2.3 Matrices

Concept: 2-dimensional, homogeneous data structures organized by rows and columns.

  • Created using matrix() .
  • Arguments: data, nrow , ncol , byrow .
  • Dimensions accessed with dim() .
  • cbind() and rbind() combine matrices/vectors.

Examples: 

# Create a matrix filled by column (default)
mat1 <- matrix(1:6, nrow = 2, ncol = 3)
# Output:
# [,1] [,2] [,3]
# [1,] 1 3 5
# [2,] 2 4 6

# Create a matrix filled by row
mat2 <- matrix(1:6, nrow = 2, ncol = 3, byrow = TRUE)
# Output:
# [,1] [,2] [,3]
# [1,] 1 2 3
# [2,] 4 5 6

dim(mat1) # 2 3 (rows, columns)

2.4 Factors

Concept: Special vectors used to store categorical data with levels representing unique categories.

  • Created using factor() .
  • Useful for statistical modeling.
  • levels() retrieves levels, nlevels() retrieves number of levels.
  • Can be ordered or unordered.

Examples: 

gender <- factor(c("Male", "Female", "Male", "Female", "Male"))
print(gender)
# Output: [1] Male Female Male Female Male
# Levels: Female Male

levels(gender) # "Female" "Male"
nlevels(gender) # 2

# Ordered factor
rating <- factor(c("Good", "Bad", "Good", "Excellent"),
  levels = c("Bad", "Good", "Excellent"),
  ordered = TRUE)
print(rating)

2.5 Missing Values

Concept: NA represents missing values in R.

  • Operations involving NA typically result in NA .
  • Use is.na() to check for missing values.
  • Many functions have na.rm = TRUE to remove NA before calculation.

Examples: 

data_with_na <- c(10, 20, NA, 30, NA)
is.na(data_with_na) # FALSE FALSE TRUE FALSE TRUE

mean(data_with_na) # NA
mean(data_with_na, na.rm = TRUE) # 20

2.6 Data Frames

Concept: The most important data structure for tabular data in R. Each column can be a different data type, but all columns have the same number of rows.

  • Created using data.frame() .
  • nrow() , ncol() , dim() give dimensions.
  • colnames() , rownames() for names.
  • str() provides structure summary.
  • summary() provides statistical summary.

Examples: 

students_df <- data.frame(
  ID = c(101, 102, 103),
  Name = c("Alice", "Bob", "Charlie"),
  Score = c(85, 92, 78),
  IsGraduated = c(TRUE, FALSE, TRUE)
)
print(students_df)
# Output:
#   ID    Name Score IsGraduated
# 1 101   Alice    85        TRUE
# 2 102     Bob    92       FALSE
# 3 103 Charlie    78        TRUE

nrow(students_df) # 3
ncol(students_df) # 4
colnames(students_df) # "ID" "Name" "Score" "IsGraduated"

2.7 Names Attribute

Concept: Assign names to elements of vectors, lists, and columns/rows of data frames/matrices for more readable and robust access.

  • Set during creation or afterwards using names() , colnames() , rownames() .
  • Provides readable access instead of numeric indices.

Examples: 

# Vector with names
ages <- c(Alice = 25, Bob = 30, Charlie = 22)
names(ages) # "Alice" "Bob" "Charlie"
ages["Bob"] # 30

# List with names
my_params <- list(mean = 10, sd = 2)
names(my_params) # "mean" "sd"
my_params$mean # 10

# Data frame column names
colnames(students_df) <- c("StudentID", "StudentName", "FinalScore", "Graduated")

2.8 Reading Tabular Data

Concept: Importing data from external files (CSV, TXT) into R.

  • read.table() : General function for reading tabular data.
  • read.csv() : Wrapper for CSV files.
  • read.delim() : Wrapper for tab-separated files.
  • Use stringsAsFactors = FALSE to avoid automatic factor conversion.

Examples: 

# Assume 'my_data.csv' exists in your working directory:
# ID,Name,Age
# 1,Alice,25
# 2,Bob,30

# Reading a CSV file
data_csv <- read.csv("my_data.csv", stringsAsFactors = FALSE)
print(data_csv)

# Reading a tab-separated file (e.g., 'my_data.txt' with tabs)
# 1\tAlice\t25
# 2\tBob\t30
data_txt <- read.table("my_data.txt", header = TRUE, sep = "\t", stringsAsFactors = FALSE)
print(data_txt)

2.9 Reading Large Tables

Concept: For very large files, read.table() can be slow and memory intensive.

  • Specify colClasses to speed up reading by pre-allocating memory.
  • Specify nrows to read only a subset for testing.
  • Use specialized packages like data.table or readr for faster reading.

Examples: 

# If 'large_data.csv' has columns: ID (integer), Name (character), Value (numeric)
initial <- read.csv("large_data.csv", nrows = 5) # Read first 5 rows to guess types
classes <- sapply(initial, class) # Get their classes
# classes might be c("integer", "character", "numeric")

large_data <- read.csv("large_data.csv", colClasses = classes)

Module 3: Subsetting

3.1 Basic Subsetting (Atomic Vectors, Lists, Matrices, Data Frames)

Concept: Extracting specific elements or subsets of R objects using various operators.

  • [ operator: General subsetting, returns same class object.
  • Integer indices: positive (select), negative (exclude), zero (empty).
  • Logical indices: TRUE/FALSE vector selects elements.
  • Named indices: select by names.

Examples: 

my_vector <- c("a", "b", "c", "d", "e")
my_vector[1] # "a" (first element)
my_vector[c(1, 3)] # "a" "c" (first and third)
my_vector[-2] # "a" "c" "d" "e" (all except second)
my_vector[c(TRUE, FALSE, TRUE, FALSE, TRUE)] # "a" "c" "e" (logical)

names(my_vector) <- c("one", "two", "three", "four", "five")
my_vector["three"] # "c" (named)

3.2 Subsetting Lists

Concept: Lists require special attention for subsetting due to their heterogeneous nature.

  • [ returns a list subset.
  • [[ extracts a single element (actual object inside).
  • $ shortcut for [[ by name.

Examples: 

my_list <- list(
  vec = c(1, 2, 3),
  char_val = "hello",
  df = data.frame(X = 1:2, Y = c("A", "B"))
)

# Using [
my_list[1] # list(vec = 1:3)
my_list["vec"] # list(vec = 1:3)

# Using [[
my_list[[1]] # 1 2 3
my_list[["vec"]] # 1 2 3

# Using $
my_list$char_val # "hello"
my_list$df # data frame itself

3.3 Subsetting Matrices

Concept: Subsetting matrices uses [row_index, col_index] .

  • Can use integer, logical, or named indices.
  • drop = TRUE (default) drops dimensions for single row/column.
  • drop = FALSE preserves dimensions.

Examples: 

my_matrix <- matrix(1:9, nrow = 3, byrow = TRUE,
  dimnames = list(c("R1", "R2", "R3"), c("C1", "C2", "C3")))

# Matrix:
#    C1 C2 C3
# R1  1  2  3
# R2  4  5  6
# R3  7  8  9

my_matrix[2, 3] # 6
my_matrix[1, ] # 1 2 3 (vector, drop=TRUE)
my_matrix[, 2] # 2 5 8 (vector)
my_matrix[c(1, 3), c(1, 3)] # subset matrix
my_matrix[1, , drop = FALSE] # 1x3 matrix
my_matrix["R2", "C3"] # 6

3.4 Subsetting Data Frames

Concept: Data frames behave like lists for column selection and like matrices for row/column selection.

  • df[row, col] for matrix-like subsetting.
  • df$column_name or df[["column_name"]] extracts a vector.
  • df["column_name"] returns a single-column data frame.
  • Logical subsetting for filtering rows.

Examples: 

data_df <- data.frame(
  Name = c("A", "B", "C", "D"),
  Age = c(20, 22, 25, 20),
  Score = c(90, 85, 95, 70)
)

data_df[1, 2] # 20 (Age of first person)
data_df$Score # 90 85 95 70 (vector)
data_df[["Name"]] # "A" "B" "C" "D" (vector)
data_df["Age"] # data frame with Age column

# Logical subsetting (filter rows)
young_students <- data_df[data_df$Age < 25, ]
# Output:
#   Name Age Score
# 1    A  20    90
# 2    B  22    85
# 4    D  20    70

3.5 Partial Matching

Concept: When using $ or [[ ]] with names, R might perform partial matching if the full name isn't provided and the partial string is unique.

  • Only for named elements.
  • Multiple matches cause errors.
  • Use options(warnPartialMatchDollar = TRUE) to warn about partial matches.

Examples: 

my_list <- list(long_name = 10, other_name = 20)
my_list$long # 10 (partial match)
my_list$l # 10 (partial match)

3.6 Removing Missing Values

Concept: Handling NA values in subsetting to obtain complete cases.

  • is.na(x) : Logical vector TRUE where NA .
  • complete.cases(...) : Logical vector indicating rows/elements with no NA .
  • Common pattern: data[complete.cases(data), ] removes rows with NA .

Examples: 

data_vec <- c(1, 2, NA, 4, NA, 5)
clean_vec <- data_vec[!is.na(data_vec)] # Removes NAs
print(clean_vec) # 1 2 4 5

data_df_na <- data.frame(
  A = c(1, 2, NA),
  B = c(NA, 4, 5),
  C = c(7, 8, 9)
)
print(data_df_na)
#    A  B C
# 1  1 NA 7
# 2  2  4 8
# 3 NA  5 9

complete_rows <- complete.cases(data_df_na)
print(complete_rows) # FALSE TRUE FALSE

clean_df <- data_df_na[complete_rows, ] # Only row 2 is complete
print(clean_df)
#   A B C
# 2 2 4 8

3.7 Vectorized Operations

Concept: R performs operations on entire vectors or arrays at once, which is faster and more concise than loops.

  • Arithmetic and comparison operators are vectorized.
  • Many built-in functions are vectorized.
  • Recycling: shorter vectors are recycled to match longer ones.

Examples: 

vec1 <- c(1, 2, 3)
vec2 <- c(10, 20, 30)
vec1 + vec2 # 11 22 33

vec1 * 2 # 2 4 6
vec1 > 1 # FALSE TRUE TRUE

# Recycling example
vec_long <- c(1, 2, 3, 4, 5, 6)
vec_short <- c(10, 20)
vec_long + vec_short # 11 22 13 24 15 26 (10, 20 recycled)

Module 4: Control Structures & Functions

4.1 Control Structures

Concept: Statements that control the flow of execution in your R code based on conditions or iterations.

  • Curly braces {} define code blocks.
  • if-else for conditional execution.
  • for loops iterate over sequences.
  • while loops repeat while condition is TRUE.
  • repeat loops infinitely until break .
  • next skips current iteration.
  • break exits loop.

Examples: 

# If-Else
score <- 75
if (score >= 90) {
  grade <- "A"
} else if (score >= 80) {
  grade <- "B"
} else {
  grade <- "C"
}
print(grade) # "C"

# For loop
for (i in 1:5) {
  print(paste("Number:", i))
}

# While loop
count <- 0
while (count < 3) {
  print(paste("Count is:", count))
  count <- count + 1
}

# Next and Break
for (num in 1:5) {
  if (num == 3) {
    next # Skip 3
  }
  if (num == 5) {
    break # Exit when 5 is reached
  }
  print(num)
}

4.2 Functions: User-Defined & Anonymous

Concept: Functions are reusable blocks of code that perform specific tasks, promoting modularity and readability.

  • Syntax: function_name <- function(arg1, arg2, ...) { body } .
  • Arguments can have default values.
  • Return value is last evaluated expression or explicit return() .
  • Anonymous functions have no name, used on-the-fly.

Examples: 

# User-Defined Function
calculate_avg <- function(numbers) {
  if (length(numbers) == 0) {
    warning("Input vector is empty.")
    return(NA)
  }
  return(sum(numbers) / length(numbers))
}
calculate_avg(c(10, 20, 30)) # 20
calculate_avg(c()) # Returns NA, prints warning

# Function with default argument
greet <- function(name = "Guest") {
  print(paste("Hello,", name))
}
greet("Alice") # "Hello, Alice"
greet() # "Hello, Guest"

# Anonymous Function (conceptual)
# function(x) { x * 2 }
# Used like: lapply(list_of_numbers, function(x) x * 2)

Module 5: Scoping Rules & Dates/Times

5.1 Scoping Rules - Symbol Binding, R Scoping Rules

Concept: Scoping rules determine how R finds the value of a symbol when used in code, especially within functions. R uses lexical scoping.

  • Lexical scoping: R looks for variable value in environment where function was defined.
  • Search path: function's environment, parent environment, global environment, loaded packages.
  • Variables inside functions are local and disappear after execution.

Examples: 

# Example 1: Local variable takes precedence
y <- 10 # Global environment
f <- function(x) {
  y <- 20 # local y
  print(x + y)
}
f(5) # 25
print(y) # 10

# Example 2: Lexical scoping in nested functions
make.power <- function(n) {
  pow <- function(x) {
    return(x^n) # 'n' found in enclosing environment
  }
  return(pow)
}
cube <- make.power(3)
cube(2) # 8
square <- make.power(2)
square(3) # 9

5.2 Optimization Example / Coding Standards

Concept: Writing efficient and readable R code improves maintainability and collaboration.

  • Prefer vectorization over loops.
  • Pre-allocate vectors when loops are necessary.
  • Use clear variable and function names.
  • Use comments and proper indentation.

Examples: 

# BAD (slow for large n):
sum_sq_loop <- function(n) {
  res <- c() # growing vector dynamically
  for (i in 1:n) {
    res <- c(res, i^2)
  }
  return(sum(res))
}

# GOOD (vectorized):
sum_sq_vectorized <- function(n) {
  return(sum((1:n)^2))
}

# GOOD (pre-allocation):
sum_sq_preallocate <- function(n) {
  res <- numeric(n)
  for (i in 1:n) {
    res[i] <- i^2
  }
  return(sum(res))
}

5.3 Dates and Times

Concept: R has specialized classes for handling dates and times to perform calculations, formatting, and comparisons.

  • Date class stores dates without time.
  • POSIXct stores date and time as seconds since 1970-01-01 UTC.
  • POSIXlt stores date and time as list components.
  • Functions: Sys.Date() , Sys.time() , as.Date() , as.POSIXct() , strptime() , format() .
  • Format codes: %Y , %m , %d , %H , %M , %S , %b , %B .

Examples: 

# Current date and time
today <- Sys.Date()
now <- Sys.time()
print(today)
print(now)

# Convert character to Date
date_str1 <- "2023-10-27"
date_obj1 <- as.Date(date_str1)
print(date_obj1)

date_str2 <- "15/01/2024"
date_obj2 <- as.Date(date_str2, format = "%d/%m/%Y")
print(date_obj2)

# Convert character to POSIXct
datetime_str1 <- "2023-10-27 14:30:00"
datetime_obj1 <- as.POSIXct(datetime_str1)
print(datetime_obj1)

# Using strptime for complex formats
custom_datetime_str <- "Oct 27, 2023 2:30 PM"
custom_datetime_obj <- strptime(custom_datetime_str, "%b %d, %Y %I:%M %p")
print(custom_datetime_obj)

# Calculations with dates/times
date1 <- as.Date("2023-01-01")
date2 <- as.Date("2023-01-10")
diff_days <- date2 - date1
print(diff_days) # Time difference of 9 days

Module 6: Loop Functions, Debugging & Simulation

6.1 Loop Functions (Apply Family)

Concept: Higher-order functions that apply a function over elements of lists, vectors, or margins of arrays/matrices, reducing explicit loops.

  • lapply(X, FUN, ...) : Applies FUN to each element of list X, returns list.
  • sapply(X, FUN, ...) : Simplifies output to vector or matrix if possible.
  • apply(X, MARGIN, FUN, ...) : Applies FUN over rows (1) or columns (2) of matrix/array.
  • mapply(FUN, ...) : Multivariate sapply, applies FUN to multiple arguments.
  • tapply(X, INDEX, FUN, ...) : Applies FUN to subsets of vector X defined by factors.
  • split(x, f) : Splits vector or data frame into groups based on factor f.

Examples: 

my_list_num <- list(a = c(1,2), b = c(3,4,5))

# lapply
lapply(my_list_num, sum)
# list(a = 3, b = 12)

# sapply
sapply(my_list_num, sum)
# a  b
# 3 12

my_matrix <- matrix(1:9, nrow = 3)
# apply sum of rows
apply(my_matrix, 1, sum) # 12 15 18
# apply mean of columns
apply(my_matrix, 2, mean) # 2 5 8

# mapply
mapply(sum, 1:3, 10:12) # 11 13 15

# tapply and split
data_values <- c(10, 12, 15, 11, 20, 18)
categories <- factor(c("A", "B", "A", "B", "A", "B"))
tapply(data_values, categories, mean)
# A: 15, B: 13.6667

split_data <- split(data_values, categories)
print(split_data)
# $A
# [1] 10 15 20
# $B
# [1] 12 11 18

6.2 Debugging Tools

Concept: Tools and techniques to identify and fix errors in R code.

  • traceback() : Shows call stack after error.
  • debug(function_name) : Enters debug mode on function call.
  • browser() : Pauses execution at point.
  • recover() : Post-mortem debugger to navigate call stack.
  • stop() : Generates error and stops execution.
  • warning() : Prints warning but continues.
  • message() : Prints informational message.

Examples: 

my_buggy_function <- function(x, y) {
  z <- x / y # error if y = 0
  print(z)
}

# Example of traceback
# my_buggy_function(10, 0) # error
# traceback() # shows error location

# Example of debug
debug(my_buggy_function)
# my_buggy_function(10, 2) # enters debug mode

# Example of browser()
another_function <- function(data) {
  if (!is.numeric(data)) {
    stop("Data must be numeric.")
  }
  browser() # pause here
  mean_val <- mean(data)
  return(mean_val)
}
# another_function(c(1,2,3)) # pauses at browser()
# another_function("hello") # stops with error

6.4 Simulation

Concept: Generating random numbers or data based on probability distributions for modeling and simulations.

  • set.seed() for reproducibility.
  • Random generators: rnorm() , runif() , rpois() , rbinom() .
  • sample() for random sampling.

Examples: 

# Reproducibility
set.seed(42)

# 5 random normal numbers (mean=0, sd=1)
norm_samples <- rnorm(5)
print(norm_samples)

# 3 random uniform numbers between 0 and 10
unif_samples <- runif(3, min = 0, max = 10)
print(unif_samples)

# Simulate rolling a die 10 times
die_rolls <- sample(1:6, 10, replace = TRUE)
print(die_rolls)

# Randomly select 2 names without replacement
names <- c("Alice", "Bob", "Charlie", "David")
selected_names <- sample(names, 2, replace = FALSE)
print(selected_names)

6.5 R Profiler

Concept: Tool to analyze performance of R code by measuring time spent in functions to identify bottlenecks.

  • Rprof() starts profiling.
  • Rprof(NULL) stops profiling.
  • summaryRprof() summarizes profiling data.
  • Optimize functions consuming most time.
