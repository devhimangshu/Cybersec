CyberSec Notes

Chapter 01: Fundamentals

Important Notes with Full Forms and Examples:

Data Communication and Networking: This is about how information moves from one place to another using networks.

Example: When you send a message on WhatsApp, your phone (sender) sends data through internet cables/Wi-Fi (media) to a server, and then to your friend's phone (receiver), following certain rules (protocols).

Network Reference Models

These are frameworks that describe how network communication works. They help different systems communicate.

Open Systems Interconnection Model

This model has seven layers, each with a specific job. Think of it like a step-by-step process for sending and receiving data.

Physical Layer

Deals with the actual cables, Wi-Fi signals, and connectors.

Example: Ethernet cables, Wi-Fi radio waves.

Data Link Layer

Manages how data moves between devices on the same network. It handles Media Access Control addresses.

Example: A switch uses MAC addresses to send data to the correct device in a local network.

Network Layer

Handles routing of data packets across different networks. It uses Internet Protocol addresses.

Example: A router uses IP addresses to send your data from your home network to Google's servers.

Transport Layer

Ensures reliable, end-to-end delivery of data. It breaks data into segments and reassembles them. Transmission Control Protocol and User Datagram Protocol operate here.

Example: When you download a file, TCP ensures all parts of the file arrive correctly.

Session Layer

Manages communication sessions between applications, like starting, managing, and ending conversations.

Example: When you log into a banking website, the session layer helps maintain your connection for that specific transaction.

Presentation Layer

Translates data into a format that the application layer can understand. Handles encryption and decryption.

Example: Converts data formats like JPEG images or MP3 audio so your computer can display or play them.

Application Layer

Provides network services to user applications.

Example: Web browsers (HTTP/HTTPS), email clients (SMTP, POP3).

Transmission Control Protocol / Internet Protocol Model

A simpler, more practical model with four layers, widely used for the internet. It maps to the OSI model.

  • Network Access Layer: Combines OSI's Physical and Data Link layers.
  • Internet Layer: Corresponds to OSI's Network Layer (uses Internet Protocol).
  • Transport Layer: Corresponds to OSI's Transport Layer (uses Transmission Control Protocol/User Datagram Protocol).
  • Application Layer: Combines OSI's Session, Presentation, and Application layers.

Three-Way Handshake (Transmission Control Protocol)

A three-step process to establish a reliable connection before data is sent using Transmission Control Protocol.

Step 1 (Synchronize)

Your computer sends a "Synchronize" message to a server to initiate a connection.

Step 2 (Synchronize-Acknowledge)

The server receives Synchronize and sends back a "Synchronize-Acknowledge" message, acknowledging your Synchronize and sending its own Synchronize to you.

Step 3 (Acknowledge)

Your computer sends an "Acknowledge" message back to the server, acknowledging its Synchronize-Acknowledge. Now the connection is established.

Example: Like two people saying "Hello?" "Hello, I hear you!" "Okay, let's talk."

Transmission Control Protocol Flags

Synchronize

Used to initiate a connection.

Acknowledge

Confirms receipt of data.

Finish

Used to terminate a connection.

Reset

Abruptly terminates a connection.

Push

Forces data to be delivered immediately.

Urgent

Indicates urgent data.

Network Address Translation

A method used by routers to allow multiple devices on a private network to share a single public Internet Protocol address when connecting to the internet.

Purpose: Saves public IP addresses and adds a layer of security by hiding internal IP addresses.

Example: Your router takes requests from your phone (private IP 192.168.1.10) and your laptop (private IP 192.168.1.11) and sends them to the internet using your home's single public IP address.

Network Transmission Media

The physical pathways through which data travels.

Guided Media

Data travels through a physical cable.

  • Twisted Pair Cable: Common for Ethernet networks. (Example: The blue cable connecting your computer to the router).
  • Coaxial Cable: Used for cable TV and some older networks. (Example: The cable connecting your TV to the wall socket).
  • Fiber Optic Cable: Uses light pulses, very fast and long distance. (Example: Modern high-speed internet connections).
Unguided Media (Wireless)

Data travels through the air.

  • Radio Waves: Used for Wi-Fi, Bluetooth. (Example: Your phone connecting to a Wi-Fi router).
  • Microwaves: Used for long-distance communication, satellite communication.
  • Infrared: Used for short-range communication, like TV remotes.

Network Devices

Hub

Connects multiple devices and broadcasts all incoming data to all connected devices. (Older, inefficient).

Switch

More intelligent than a hub, sends data only to the intended recipient using Media Access Control addresses.

Router

Connects different networks and forwards data packets between them using Internet Protocol addresses.

Bridge

Connects two separate network segments, often used to extend a network.

Repeater

Boosts a signal to extend its range.

Gateway

A device that connects two different networks that use different protocols.

Modulator-Demodulator

Converts digital signals from your computer into analog signals for transmission over telephone lines or cable, and vice versa.

Information Security Definition

Protecting information and information systems from unauthorized access, use, disclosure, disruption, modification, or destruction.

Example: Keeping your personal photos private, ensuring your online banking transactions are accurate, and being able to access your email whenever you need it.

Information Security Goals (Confidentiality, Integrity, Availability Triad)

  • Confidentiality: Keeping sensitive information private and preventing unauthorized disclosure.

    Example: Encrypting your emails so only the intended recipient can read them.

  • Integrity: Ensuring that information is accurate, complete, and has not been altered without authorization.

    Example: Making sure your bank balance displayed online is correct and hasn't been tampered with.

  • Availability: Ensuring that authorized users can access information and systems when needed.

    Example: A website always being accessible to its users, not crashing or being offline.

Basic Concepts of Cryptography

The practice and study of techniques for secure communication in the presence of adversaries.

Encryption

Converting plaintext (readable data) into ciphertext (unreadable, coded data) to protect it.

Decryption

Converting ciphertext back into plaintext.

Keys

Secret values used in encryption and decryption.

Symmetric-key Cryptography

Uses the same key for both encryption and decryption.

Example: Advanced Encryption Standard.

Asymmetric-key Cryptography (Public-key Cryptography)

Uses a pair of keys: a public key for encryption and a private key for decryption.

Example: Rivest–Shamir–Adleman used in HTTPS.

Hashing

A one-way mathematical function that converts data into a fixed-size string of characters (a hash value). It's used to verify data integrity. You cannot get the original data back from a hash.

Example: If you download a software file, its hash value can be compared to a known good hash to ensure the file hasn't been tampered with.

Steganography

The art and science of hiding the existence of a message, rather than just scrambling the message itself.

Example: Hiding a secret text message within an image file, so to a casual observer, it just looks like a picture.

Chapter 02: Hacking Concepts & Cyber Law

Important Notes with Full Forms and Examples:

Hacking

Unauthorized access to or manipulation of computer systems, networks, or data.

Example: Gaining access to someone's email account without their permission.

Types of Hacking and Hackers

Black Hat Hackers

Individuals who engage in hacking with malicious intent for personal gain, destruction, or other illegal activities.

Example: Someone breaking into a company's database to steal customer credit card numbers.

White Hat Hackers (Ethical Hackers)

Security professionals who use their hacking skills to identify and fix vulnerabilities in systems, with authorization.

Example: A security consultant hired by a bank to test its online banking system for weaknesses.

Grey Hat Hackers

Individuals who find vulnerabilities without authorization but don't act maliciously.

Example: Someone finds a bug in a website, exploits it, and then informs the website owner.

Cybercrime

Criminal activities carried out using computers or the internet.

Example: Online fraud, identity theft, cyberbullying, spreading malware.

Types of Cybercrime
  • Cyberbullying: Harassing, threatening, or embarrassing someone online.
  • Cyberstalking: Repeatedly harassing or threatening someone online, often escalating to real-life threats.
  • Online Fraud: Deceptive practices conducted online to trick individuals out of money or information. (Example: Phishing scams).
  • Intellectual Property Theft: Stealing copyrighted material, patents, or trade secrets online.
  • Data Theft: Unauthorized copying or removal of data from a computer system.
  • Identity Theft (including Password Theft): Stealing and using someone's personal information for fraudulent purposes.
  • Cyberterrorism: Using computer networks to cause disruption or fear for political or ideological reasons.

Classifications of Security Attacks

Passive Attacks

Attacks that monitor or eavesdrop on network traffic without altering any data. The attacker's goal is to gain information.

Example: Snooping (secretly listening to conversations or monitoring network traffic).

Active Attacks

Attacks that modify, inject, or create data streams, or disrupt services. The attacker's goal is to alter system resources or affect their operation.

Example: Denial of Service attack, flooding a server with traffic.

Essential Terminology

  • Threat: A potential danger or possibility of an attack that could harm an asset.
  • Vulnerability: A weakness or flaw in a system that can be exploited by a threat.
  • Target of Evaluation: The specific system or component being evaluated for security.
  • Attack: An attempt to exploit a vulnerability to gain unauthorized access or cause damage.
  • Exploit: Software or technique that takes advantage of a vulnerability.

Concept of Ethical Hacking

Performing hacking activities with explicit permission to identify vulnerabilities and strengthen security.

Example: A company hires an ethical hacker to try and break into its corporate network to find security weaknesses.

Phases of Ethical Hacking

  1. Reconnaissance (Information Gathering): Collecting information about the target system.
  2. Scanning: Using tools to identify open ports, services, and vulnerabilities.
  3. Gaining Access: Exploiting vulnerabilities to enter the system.
  4. Maintaining Access: Establishing persistent presence for future access.
  5. Clearing Tracks: Removing evidence of intrusion to avoid detection.

Hacktivism

Hacking for political or social causes, combining hacking skills with activism.

Example: A group defacing a government website to protest a new law.

Cyber Law

The legal framework that governs activities in cyberspace. In India, the primary law is the Information Technology Act, 2000, with amendments.

Cyber Terrorism

The use of computer networks and the internet to carry out acts of terrorism, often involving large-scale disruption, fear, or violence.

Example: A group launching a cyberattack to disable a country's power grid, causing widespread chaos.

Offences Covered under Indian Cyber Laws

  • Hacking (Section 43, 66): Unauthorized access, damage to computer systems or data.
  • Data Theft: Illegally copying or extracting data.
  • Identity Theft (including Password Theft): Misappropriating or using someone else's identity or passwords.
  • Email Spoofing: Sending emails with a forged sender address to deceive the recipient.
  • Sending Offensive Messages: Messages that are grossly offensive or menacing.
  • Voyeurism: Capturing or transmitting images of a private area without consent.
  • Cyber Terrorism: Hacking intending to threaten India's unity, security, or to strike terror.

Punishment for Cybercrime in India

Penalties vary depending on the severity and nature of the crime, including imprisonment (from months to life) and fines (thousands to lakhs of rupees).

Chapter 03: Malware, Denial of Service, Intrusion Detection/Prevention Systems & Web Application Based Threats

Important Notes with Full Forms and Examples:

Malware (Malicious Software)

Software designed to gain unauthorized access or damage a computer system without the user's knowledge or consent.

Example: A virus infecting your computer, or ransomware locking your files.

Types of Malware
Virus

Attaches itself to legitimate programs or documents and spreads when executed. Requires human action.

Example: An infected email attachment that corrupts files when opened.

Worm

Standalone malware that self-replicates and spreads across networks without human interaction.

Example: A worm scanning a network and installing itself automatically.

Trojan Horse

Disguises as legitimate software but performs malicious actions once installed. Does not self-replicate.

Example: A free game that secretly installs a backdoor for remote access.

Spyware

Secretly gathers information about a user's activities and sends it to an attacker.

Example: Adware that tracks browsing habits.

Adware

Automatically displays or downloads unwanted advertisements, often bundled with free software.

Example: Pop-up ads appearing constantly after installing a free program.

Ransomware

Encrypts files or locks a computer, then demands a ransom for decryption or unlocking.

Example: WannaCry ransomware encrypting thousands of computers worldwide.

Types of Computer Viruses
  • File Virus: Attaches to executable files (.exe, .com).
  • Boot Sector Virus: Infects the boot sector of a hard drive or floppy disk.
  • Macro Virus: Written in macro language, infects documents (Word, Excel).
  • Email Virus: Spreads through email attachments.
  • Multi-variant Virus (Polymorphic): Changes code each time it infects a new system.
Some Indications of a Malware Attack
  • Slow computer performance.
  • Frequent crashes or freezing.
  • Unusual pop-up advertisements.
  • Missing files or corrupted data.
  • Unknown programs running or suspicious network activity.
  • Browser homepage changes without your consent.
Popular Antivirus Programs

Software designed to detect, prevent, and remove malware.

Examples: Norton, McAfee, Avast, Bitdefender, Kaspersky, Windows Defender.

Basic Idea of How Antivirus Identifies a Virus
  • Signature-based Detection: Uses a database of known virus signatures to scan files.
  • Heuristics-based Detection: Analyzes behavior of programs to flag suspicious actions.
  • Cloud-based Detection: Uses real-time data from a global network of users for threat intelligence.
VirusTotal Website

An online service that analyzes suspicious files and URLs for malware using multiple antivirus engines.

Example: Upload a suspicious email attachment to check if it is malicious before opening.

Denial of Service Attack

An attack designed to make a machine or network resource unavailable by overwhelming it with requests or traffic.

Example: Flooding a website's server with so many requests that legitimate users cannot access it.

Distributed Denial of Service Attack

A Denial of Service attack launched from multiple compromised systems simultaneously, making it harder to stop.

Example: Thousands of infected computers sending requests to a target server, bringing it down.

Intrusion Detection System

Monitors network or system activities for malicious activities or policy violations and generates alerts.

Example: Detecting multiple failed login attempts from an unusual IP address.

Intrusion Prevention System

Detects intrusions and actively takes steps to prevent them from succeeding.

Example: Blocking traffic from a known attack source immediately.

Snooping and Eavesdropping

Secretly listening to conversations or monitoring network traffic without authorization.

Example: Capturing Wi-Fi traffic to read unencrypted messages.

Keyloggers

Software or hardware that records every keystroke made on a computer.

Example: A malicious keylogger capturing usernames and passwords on a public computer.

Firewall

A network security system that monitors and controls incoming and outgoing network traffic based on security rules.

Example: Blocking unknown applications from accessing the internet.

Bots and Botnets (Zombies)

Bot: A compromised computer controlled remotely by an attacker.

Botnet: A network of bots used for coordinated malicious activities.

Example: A botnet launching a massive Denial of Service attack against a government website.

Web Application Based Threats

  • Cross-Site Scripting: Injecting malicious scripts into web pages viewed by others.
  • Structured Query Language Injection: Inserting malicious SQL statements to attack data-driven applications.
  • Command Injection: Executing arbitrary commands on the host operating system via vulnerable applications.
  • Buffer Overflow: Writing more data into a buffer than it can hold, causing crashes or code execution.
  • Directory Traversal: Accessing files outside the web server's root directory by manipulating URLs.
  • Phishing Scams: Fraudulent attempts to obtain sensitive information by impersonation.
  • Drive-by Downloads: Malicious downloads occurring without user knowledge when visiting compromised sites.

Chapter 04: Wireless Networking & Protocols/Proxy

Important Notes with Full Forms and Examples:

Wireless Networking

A type of network that uses radio waves rather than physical cables to transmit data between devices.

Example: Your home Wi-Fi network that connects your phone, laptop, and smart TV without wires.

Wireless Standards

Rules and specifications that ensure compatibility and interoperability among wireless devices.

Institute of Electrical and Electronics Engineers 802.11: Family of specifications for Wireless Local Area Networks (Wi-Fi). Versions include 802.11b, g, n, ac, ax (Wi-Fi 6).

Common Terms Used in Wireless Networking

  • Wireless Local Area Network: A local area network using radio waves instead of cables.
  • Wireless Access Point: Device allowing Wi-Fi devices to connect to a wired network.
  • Cellular: Mobile communication technology dividing areas into cells served by base stations.
  • Attenuation: Reduction in signal strength over distance or obstacles.
  • Antenna: Converts electrical power into radio waves and vice versa.
  • Microwave: High-frequency radio waves for point-to-point wireless communication.
  • Jamming: Intentional interference with wireless communications.
  • Service Set Identifier: The name of a Wi-Fi network.
  • Bluetooth: Short-range wireless technology for exchanging data.
  • Wi-Fi Hotspots: Locations offering wireless internet access.

Wireless Attacks

  • War Driving: Driving around to detect unprotected wireless networks.
  • War Walking: Walking to detect wireless networks, often in urban areas.
  • War Flying: Using drones to map wireless networks from the air.
  • War Chalking: Drawing symbols in public places to indicate wireless network presence.
  • Bluejacking: Sending unsolicited messages to nearby Bluetooth devices.

How to Secure Wireless Networks

  • Use strong passwords and encryption (WPA2 or WPA3).
  • Disable Service Set Identifier broadcast (limited security).
  • Media Access Control address filtering to allow only specific devices.
  • Enable firewall on router.
  • Keep firmware updated.
  • Use Virtual Private Network on public Wi-Fi.

Protocols and Proxy

A protocol is a set of rules governing data transmission. A port identifies a specific application or service.

Hypertext Transfer Protocol

Fundamental protocol for transferring web pages over the internet.

Default Port: 80

Hypertext Transfer Protocol Secure

Secure version of HTTP using Secure Sockets Layer or Transport Layer Security encryption.

Default Port: 443

Example: Padlock icon in browser URL bar.

File Transfer Protocol

Used for transferring files between computers.

Default Ports: 20 (data), 21 (control)

Secure Shell

Cryptographic protocol for secure remote access.

Default Port: 22

Example: Admin logging into remote server securely.

Telecommunication Network

Older protocol for remote command-line access, insecure as it sends plaintext.

Default Port: 23

Replaced largely by Secure Shell.

Simple Mail Transfer Protocol

Used for sending outgoing email messages.

Default Ports: 25, 587 (secure submission)

Domain Name System

Translates domain names into Internet Protocol addresses.

Default Port: 53

Example: google.com to 172.217.160.142

Post Office Protocol version 3

Used by email clients to retrieve email messages from a server.

Default Port: 110

Proxy Concept

A proxy server acts as an intermediary for requests from clients seeking resources from other servers.

Example: Website sees proxy's IP, not yours.

Different Types of Proxy

  • Forward Proxy: Forwards client requests to external servers, often used in corporate networks.
  • Reverse Proxy: Intercepts requests to web servers, used for load balancing and security.
  • Proxy Chain: Routing traffic through multiple proxies for enhanced anonymity.

Chapter 05: Stay Secure in the Digital World

Important Notes with Full Forms and Examples:

Usage of Password

Passwords are your primary defense for securing accounts and data.

Example: The password you use to log into your email or social media accounts.

Different Types of Password

  • Biometric Password: Uses unique biological characteristics (fingerprint, facial recognition).
  • Pattern-based Graphical Password: Uses a pattern drawn on a grid or image.

Strong Password Technique

Creating passwords that are difficult to guess or crack.

  • Length: At least 12-16 characters or more.
  • Complexity: Mix uppercase, lowercase, numbers, special characters.
  • Uniqueness: Don't reuse passwords across accounts.
  • No Personal Info: Avoid names, birthdates, pet names.
  • Phrase-based: Use memorable but long phrases.

Example: "Ilovetoreadbooks@night!2025"

Types of Password Attacks

  • Brute Force Attack: Trying every possible combination until the password is found.
  • Dictionary Attack: Trying common words and leaked passwords.
  • Rainbow Table Attack: Using pre-computed tables of hashed passwords.
  • Credential Stuffing: Using leaked username/password combos on other sites.
  • Shoulder Surfing: Observing someone entering their password.

Steps to Stay Secure in the Digital World

  1. Have Strong Passwords: Use long, complex, unique passwords. Use password managers.
  2. Encrypt Your Data: Protect files and folders with encryption tools like BitLocker or FileVault.
  3. Security Suite Software: Install antivirus, anti-malware, anti-spyware software.
  4. Firewall Setup: Configure firewalls to control network traffic.
  5. Update Operating System: Regularly install updates and patches.
  6. Enable Two-Factor Authentication: Adds extra security beyond passwords.
  7. Backup Your Data: Regularly back up important files to external or cloud storage.
  8. Be Wary of Suspicious Links and Emails: Avoid clicking unknown links or downloading attachments.
  9. Use Public Wi-Fi with Caution: Avoid sensitive activities or use Virtual Private Networks.
  10. Be Mindful of Information Shared Online: Limit personal info on social media.
