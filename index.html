Module 1: Getting Started, Background

1.1 Installing R on Windows

Concept: R is a free, open-source programming language and environment. RStudio is an Integrated Development Environment (IDE) that makes working with R much easier.

  • Download R from CRAN (Comprehensive R Archive Network): cran.r-project.org .
  • Download RStudio Desktop (Free version) from rstudio.com .
  • Install R first, then RStudio.
  • RStudio provides a console, script editor, environment pane, plot pane, etc. for a better workflow.

Example (Conceptual): You'd typically double-click the downloaded .exe files and follow the installation wizard.

1.2 Writing Code / Setting Your Working Directory (Windows)

Concept: R code can be written directly in the R Console (for single lines or testing). For multiple lines or saving your work, use an R Script file (.R extension) in RStudio's editor. The Working Directory is the default location where R reads files from and writes files to. It's crucial for managing your data and scripts.

  • In RStudio: Use File → New File → R Script to open a script editor.
  • Run code from a script: Select lines and press Ctrl+Enter (Windows) or Cmd+Enter (Mac).
  • Setting Working Directory:
    • Manually: Session → Set Working Directory → Choose Directory... (RStudio).
    • Programmatically: setwd("path/to/your/folder") . Use forward slashes / even on Windows, or double backslashes \\ .
  • Getting Working Directory: getwd() 
# Writing code in a script and running
# This is a comment
result <- 5 + 3
print(result)

# Setting working directory (Windows example)
setwd("C:/Users/YourName/Documents/RProjects")

# Getting current working directory
current_dir <- getwd()
print(current_dir)

1.3 Overview and History of R

Concept: R evolved from the S language, developed by Ross Ihaka and Robert Gentleman at the University of Auckland. It's now maintained by the R Core Team.

  • Powerful for statistical modeling, data analysis, graphics, and machine learning.
  • Open-source and highly extensible through packages (libraries of functions and data).
  • Vibrant community and extensive documentation.
  • Emphasizes vectorized operations for efficiency.

1.4 R Console Input and Evaluation

Concept: The R Console is an interactive command-line interface where you type R expressions, and R immediately evaluates them and prints the result.

  • Expressions are evaluated from left to right.
  • Assignment operator: <- (preferred) or = .
  • Case-sensitive (e.g., myVar is different from myvar ).
  • Comments start with # . 
# Simple expression evaluation
10 / 2 # Output: 5

# Variable assignment
x <- 10
y = 20 # less preferred
z <- x + y
print(z) # Output: 30

# Using functions
sqrt(25) # Output: 5

Questions & Brief Answers:

  1. Q: What is RStudio, and how does it relate to R?
    A: RStudio is an Integrated Development Environment (IDE) that provides a user-friendly interface for working with R, making coding, debugging, and project management easier. It runs on top of R.
  2. Q: How do you set your working directory in R programmatically, and why is it important?
    A: Use setwd("path/to/directory") . It's important because it tells R where to look for data files and where to save output, ensuring your project files are organized and accessible.
  3. Q: What is the primary difference between typing code directly in the R Console and using an R Script file?
    A: Console is for immediate evaluation and testing of single lines. Script files are for writing, saving, and executing multiple lines of code, enabling reproducibility and project management.
  4. Q: What symbol is used for comments in R?
    A: # .

Module 2: Data Types - R Objects and Attributes

2.1 R Objects and Attributes

Concept: In R, everything is an object. Objects have attributes, which are metadata describing the object (e.g., names, dimensions, class).

  • class() : Returns the class of an object (its type).
  • typeof() : Returns the type of storage of an object.
  • attributes() : Returns all attributes of an object. 
x <- c(1, 2, 3)
class(x) # "numeric"
typeof(x) # "double" (default numeric is double precision)
attributes(x) # NULL (atomic vectors don't have many attributes by default)

m <- matrix(1:4, nrow=2)
attributes(m) # Output includes 'dim' (dimensions)

2.2 Vectors and Lists

Concept: Atomic Vectors are homogeneous collections of elements of the same type. Lists are heterogeneous collections that can contain different types of objects.

  • Atomic Vectors types: logical, integer, numeric, complex, character, raw.
  • Lists can contain vectors, other lists, data frames, functions, etc.
  • Coercion hierarchy: logical → integer → numeric → character.
  • Accessing elements: [ ] for vectors, [ ] and [[ ]] for lists. 
# Atomic Vectors
log_vec <- c(TRUE, FALSE, TRUE)
int_vec <- c(1L, 2L, 3L)
num_vec <- c(1.5, 2.0, 3.5)
char_vec <- c("apple", "banana", "cherry")

# Coercion example
mixed_vec <- c(1, "hello", TRUE)
class(mixed_vec) # "character" (all coerced to character)

# Lists
my_nested_list <- list(
  num_data = c(10, 20, 30),
  text_data = "sample text",
  is_active = TRUE,
  another_list = list(sub_item = 5)
)

2.3 Matrices

Concept: 2-dimensional, homogeneous data structures organized by rows and columns.

  • Created using matrix().
  • Arguments: data, nrow, ncol, byrow (default FALSE).
  • Dimensions accessed with dim() .
  • cbind() and rbind() combine matrices/vectors. 
# Create a matrix filled by column (default)
mat1 <- matrix(1:6, nrow = 2, ncol = 3)
# Output:
#      [,1] [,2] [,3]
# [1,]    1    3    5
# [2,]    2    4    6

# Create a matrix filled by row
mat2 <- matrix(1:6, nrow = 2, ncol = 3, byrow = TRUE)
# Output:
#      [,1] [,2] [,3]
# [1,]    1    2    3
# [2,]    4    5    6

dim(mat1) # 2 3 (rows, columns)

2.4 Factors

Concept: Special vectors used to store categorical data with levels representing unique categories.

  • Created using factor() .
  • Useful for statistical modeling.
  • levels() retrieves levels, nlevels() retrieves number of levels.
  • Can be ordered or unordered. 
gender <- factor(c("Male", "Female", "Male", "Female", "Male"))
print(gender)
# Output: [1] Male Female Male Female Male
# Levels: Female Male

levels(gender) # "Female" "Male"
nlevels(gender) # 2

# Ordered factor
rating <- factor(c("Good", "Bad", "Good", "Good", "Excellent"),
                 levels = c("Bad", "Good", "Excellent"),
                 ordered = TRUE)
print(rating)

2.5 Missing Values

Concept: NA represents missing values.

  • Any operation involving NA typically results in NA .
  • Use is.na() to check for NA values.
  • Many functions have an na.rm = TRUE argument to remove NA s before calculation. 
data_with_na <- c(10, 20, NA, 30, NA)
is.na(data_with_na) # FALSE FALSE TRUE FALSE TRUE

mean(data_with_na) # NA
mean(data_with_na, na.rm = TRUE) # 20

2.6 Data Frames

Concept: The most important data structure for tabular data in R. Each column can be a different data type, but all columns must have the same number of rows.

  • Created using data.frame() .
  • nrow() , ncol() , dim() give dimensions.
  • colnames() , rownames() for names.
  • str() provides a summary of the structure.
  • summary() provides statistical summary. 
students_df <- data.frame(
  ID = c(101, 102, 103),
  Name = c("Alice", "Bob", "Charlie"),
  Score = c(85, 92, 78),
  IsGraduated = c(TRUE, FALSE, TRUE)
)
print(students_df)
# Output:
#   ID    Name Score IsGraduated
# 1 101   Alice    85        TRUE
# 2 102     Bob    92       FALSE
# 3 103 Charlie    78        TRUE

nrow(students_df) # 3
ncol(students_df) # 4
colnames(students_df) # "ID" "Name" "Score" "IsGraduated"

2.7 Names Attribute

Concept: Allows you to assign names to elements of vectors, lists, and columns/rows of data frames/matrices.

  • Can be set during creation or afterwards using names() , colnames() , rownames() .
  • Provides more readable and robust access to data than numeric indices. 
# Vector with names
ages <- c(Alice = 25, Bob = 30, Charlie = 22)
names(ages) # "Alice" "Bob" "Charlie"
ages["Bob"] # 30

# List with names
my_params <- list(mean = 10, sd = 2)
names(my_params) # "mean" "sd"
my_params$mean # 10

# Data frame column names are automatically set from creation
# or can be set with colnames()
colnames(students_df) <- c("StudentID", "StudentName", "FinalScore", "Graduated")

2.8 Reading Tabular Data

Concept: Importing data from external files (e.g., CSV, TXT) into R.

  • read.table() : General function for reading tabular data.
  • read.csv() : Wrapper for read.table() with comma delimiter and header by default.
  • read.delim() : Wrapper for tab-separated files. 
# Assume 'my_data.csv' exists in your working directory:
# ID,Name,Age
# 1,Alice,25
# 2,Bob,30

# Reading a CSV file
data_csv <- read.csv("my_data.csv", stringsAsFactors = FALSE)
print(data_csv)

# Reading a tab-separated file (e.g., 'my_data.txt' with tabs)
# 1\tAlice\t25
# 2\tBob\t30
data_txt <- read.table("my_data.txt", header = TRUE, sep = "\t", stringsAsFactors = FALSE)
print(data_txt)

2.9 Reading Large Tables

Concept: For very large files, read.table() can be slow and memory intensive.

  • Specify colClasses argument to speed up reading by pre-allocating memory and avoiding type guessing.
  • Specify nrows to read only a certain number of rows (useful for testing or partial reads).
  • Specialized packages like data.table ( fread() ) or readr ( read_csv() ) are much faster for large datasets. 
# If 'large_data.csv' has columns: ID (integer), Name (character), Value (numeric)
initial <- read.csv("large_data.csv", nrows = 5) # Read first 5 rows to guess types
classes <- sapply(initial, class) # Get their classes
# classes might be: c("integer", "character", "numeric") # Manually define if known

large_data <- read.csv("large_data.csv", colClasses = classes)

Questions & Brief Answers:

  1. Q: What is the fundamental difference in homogeneity between a vector and a data frame?
    A: A vector is homogeneous (all elements must be of the same type), while a data frame is heterogeneous (columns can be of different types, though elements within a column are of the same type).
  2. Q: You have my_vec <- c(10, "20", TRUE) . What will class(my_vec) return and why?
    A: "character" because R coerces all elements to the most flexible type when mixed, which is character in this case.
  3. Q: How do you create a matrix named m with 2 rows and 3 columns, filled by row with numbers 1 to 6?
    A: m <- matrix(1:6, nrow = 2, ncol = 3, byrow = TRUE) .
  4. Q: What are NA values in R, and how do you handle them in a sum calculation?
    A: NA represents missing values. To sum, use sum(vector_name, na.rm = TRUE) .
  5. Q: Describe the data.frame structure. What is its common use?
    A: A data frame is a 2D tabular structure where each column can have a different data type, but all columns have the same number of rows. It's commonly used for storing datasets, similar to a spreadsheet.
  6. Q: How do you read a CSV file named "students.csv" with a header row into a data frame called student_data , ensuring string columns are NOT converted to factors?
    A: student_data <- read.csv("students.csv", header = TRUE, stringsAsFactors = FALSE) .

Module 3: Subsetting

3.1 Basic Subsetting (Atomic Vectors, Lists, Matrices, Data Frames)

Concept: Extracting specific elements or subsets of R objects using various operators.

  • [ operator: Most general subsetting operator. Can take integer, logical, or character vectors. Always returns an object of the same class as the original (unless drop = TRUE is used with matrices/data frames and a single dimension is selected).
  • Integer indices: Positive integers (select positions), negative integers (exclude positions), zero (returns empty object).
  • Logical indices: A logical vector (TRUE/FALSE) of the same length, selecting elements where TRUE.
  • Named indices (character vector): Selects elements by their names. 
my_vector <- c("a", "b", "c", "d", "e")
my_vector[1] # "a" (first element)
my_vector[c(1, 3)] # "a" "c" (first and third)
my_vector[-2] # "a" "c" "d" "e" (all except second)
my_vector[c(TRUE, FALSE, TRUE, FALSE, TRUE)] # "a" "c" "e" (logical)
names(my_vector) <- c("one", "two", "three", "four", "five")
my_vector["three"] # "c" (named)

3.2 Subsetting Lists

Concept: Lists require special attention for subsetting due to their heterogeneous nature.

  • [ operator: Returns a list (subset of the original list), even if it contains only one element. Preserves the list structure.
  • [[ operator: Extracts a single element from a list. The result is the actual object inside that list element.
  • $ operator: A convenient shortcut for [[ when extracting a single element by name. 
my_list <- list(
  vec = c(1, 2, 3),
  char_val = "hello",
  df = data.frame(X = 1:2, Y = c("A", "B"))
)

# Using [
my_list[1]
my_list["vec"] # Output: list(vec = 1:3) - still a list

# Using [[
my_list[[1]]
my_list[["vec"]] # Output: 1 2 3 - the vector itself

# Using $
my_list$char_val # Output: "hello"
my_list$df       # Output: the data frame itself

3.3 Subsetting Matrices

Concept: Subsetting matrices uses [row_index, col_index] .

  • Can use integer, logical, or named indices for rows and/or columns.
  • drop = TRUE (default): If you select a single row or column, the result is "dropped" to the lowest possible dimension (e.g., a vector).
  • drop = FALSE : Preserves the dimension, even if selecting a single row/column (returns a 1-row/1-column matrix). 
my_matrix <- matrix(1:9, nrow = 3, byrow = TRUE,
                    dimnames = list(c("R1", "R2", "R3"), c("C1", "C2", "C3")))
# Output:
#    C1 C2 C3
# R1  1  2  3
# R2  4  5  6
# R3  7  8  9

my_matrix[2, 3]       # Single element: 6
my_matrix[1, ]        # First row (as vector due to drop=TRUE): 1 2 3
my_matrix[, 2]        # Second column (as vector): 2 5 8
my_matrix[c(1, 3), c(1, 3)] # Subset of rows and columns
my_matrix[1, , drop = FALSE] # First row (as 1x3 matrix)
my_matrix["R2", "C3"] # Subsetting by name: 6

3.4 Subsetting Data Frames

Concept: Data frames behave like lists of vectors for column selection and like matrices for [row, col] selection.

  • df[row_indices, col_indices] behaves like matrix subsetting.
  • df$column_name or df[["column_name"]] extract a column as a vector.
  • df["column_name"] returns a single-column data frame.
  • Logical subsetting of rows is very common for filtering data. 
data_df <- data.frame(
  Name = c("A", "B", "C", "D"),
  Age = c(20, 22, 25, 20),
  Score = c(90, 85, 95, 70)
)

data_df[1, 2]       # Age of first person: 20
data_df$Score       # The Score column as a vector: 90 85 95 70
data_df[["Name"]]   # The Name column as a vector: "A" "B" "C" "D"
data_df["Age"]      # The Age column as a data frame:
#   Age
# 1  20
# 2  22
# 3  25
# 4  20

# Logical subsetting (filter rows)
young_students <- data_df[data_df$Age < 25, ]
# Output:
#   Name Age Score
# 1    A  20    90
# 2    B  22    85
# 4    D  20    70

3.5 Partial Matching

Concept: When using $ or [[ ]] with character strings for names, R might perform partial matching if the full name isn't provided, as long as the partial string is unique. Generally, avoid relying on this for clarity and robustness.

  • Only for named elements.
  • If multiple names match the partial string, it's an error.
  • options(warnPartialMatchDollar = TRUE) can warn about this. 
my_list <- list(long_name = 10, other_name = 20)
my_list$long  # 10 (partial match)
my_list$l     # 10 (partial match)

# Example causing error due to ambiguity
list_ambig <- list(item1 = 1, item2 = 2)
# list_ambig$item # Error: object 'item' not found

3.6 Removing Missing Values

Concept: Handling NA values in subsetting to obtain complete cases.

  • is.na(x) : Returns a logical vector, TRUE where x is NA .
  • complete.cases(...) : Returns a logical vector indicating which rows/elements have no NA s.
  • Common pattern: data[complete.cases(data), ] to remove rows with NA s from a data frame. 
data_vec <- c(1, 2, NA, 4, NA, 5)
clean_vec <- data_vec[!is.na(data_vec)] # Removes NAs
print(clean_vec) # 1 2 4 5

data_df_na <- data.frame(
  A = c(1, 2, NA),
  B = c(NA, 4, 5),
  C = c(7, 8, 9)
)
print(data_df_na)
#    A  B C
# 1  1 NA 7
# 2  2  4 8
# 3 NA  5 9

complete_rows <- complete.cases(data_df_na)
print(complete_rows) # FALSE TRUE FALSE

clean_df <- data_df_na[complete_rows, ] # Only row 2 is complete
print(clean_df)
#   A B C
# 2 2 4 8

3.7 Vectorized Operations

Concept: R's strength lies in performing operations on entire vectors or arrays at once, rather than element by element (which is what loops do). This is often much faster and more concise.

  • Arithmetic operations (+, -, *, /, ^) and comparison operators (>, <, ==, !=) are vectorized.
  • Many built-in functions (e.g., sum(), mean(), log(), sqrt()) are vectorized.
  • Recycling: If two vectors have different lengths, the shorter one's elements are recycled to match the longer one. If lengths are not multiples, a warning is issued. 
vec1 <- c(1, 2, 3)
vec2 <- c(10, 20, 30)
vec1 + vec2 # Element-wise addition: 11 22 33
vec1 * 2    # Multiply each element by 2: 2 4 6
vec1 > 1  # Logical comparison: FALSE TRUE TRUE

# Recycling example
vec_long <- c(1, 2, 3, 4, 5, 6)
vec_short <- c(10, 20)
vec_long + vec_short # Output: 11 22 13 24 15 26 (10, 20 recycled)

  1. Q: You have a list my_data <- list(values = c(1, 2, 3), settings = list(mode = "auto")) . How do you extract the value mode from the settings list element?

Module 4: Control Structures & Functions

4.1 Control Structures

Concept: Statements that control the flow of execution in your R code based on conditions or iterations.

  • Curly braces {} define code blocks.
  • if-else : Conditional execution.
  • for loops: Iterate over a sequence.
  • while loops: Repeat while condition is TRUE.
  • repeat : Infinite loop, must use break to exit.
  • next : Skips current iteration.
  • break : Exits loop entirely. 
# If-Else
score <- 75
if (score >= 90) {
  grade <- "A"
} else if (score >= 80) {
  grade <- "B"
} else {
  grade <- "C"
}
print(grade) # "C"

# For loop
for (i in 1:5) {
  print(paste("Number:", i))
}

# While loop
count <- 0
while (count < 3) {
  print(paste("Count is:", count))
  count <- count + 1
}

# Next and Break
for (num in 1:5) {
  if (num == 3) {
    next # Skip 3
  }
  if (num == 5) {
    break # Exit when 5 is reached
  }
  print(num)
}

4.2 Functions: User-Defined Functions, Anonymous Functions

Concept: Functions are reusable blocks of code that perform a specific task. They promote modularity, readability, and avoid code repetition.

  • Syntax: function_name <- function(arg1, arg2, ...) { body } .
  • Arguments can have default values.
  • Return value is the last evaluated expression or explicit return() .
  • Anonymous functions are unnamed and often used as arguments to higher-order functions. 
# User-Defined Function
calculate_avg <- function(numbers) {
  if (length(numbers) == 0) {
    warning("Input vector is empty.")
    return(NA)
  }
  return(sum(numbers) / length(numbers))
}
calculate_avg(c(10, 20, 30)) # 20
calculate_avg(c())           # Returns NA, prints warning

# Function with default argument
greet <- function(name = "Guest") {
  print(paste("Hello,", name))
}
greet("Alice") # "Hello, Alice"
greet()        # "Hello, Guest"

# Anonymous Function (conceptual)
# function(x) { x * 2 }
# Used like: lapply(list_of_numbers, function(x) x * 2)

Questions & Brief Answers:

  1. Q: Write an if-else statement that prints "Positive" if a variable num is greater than 0, and "Non-positive" otherwise.
    A: 
    if (num > 0) { print("Positive") } else { print("Non-positive") }
  2. Q: When is a while loop more appropriate than a for loop?
    A: When the number of iterations is unknown and depends on a condition being met.
  3. Q: What is the purpose of the next statement in a loop?
    A: It skips the current iteration and moves to the next.
  4. Q: Define a user-defined function named square_number that takes one argument x and returns its square.

    A: A function without a name, commonly used as arguments to higher-order functions like lapply , sapply .

Module 5: Scoping Rules & Dates/Times

5.1 Scoping Rules - Symbol Binding, R Scoping Rules

Concept: Scoping rules determine how R finds the value of a symbol (variable name) when it's used in code, especially within functions. R uses lexical scoping.

  • Lexical Scoping: R looks for a variable's value in the environment where the function was defined (its parent environment), not where it was called.
  • Search Path: function's own environment → enclosing environment → global environment → namespaces of loaded packages.
  • Variables defined inside a function are local to that function and disappear after execution. 
# Example 1: Local variable takes precedence
y <- 10 # Global environment
f <- function(x) {
  y <- 20 # 'y' local to function f
  print(x + y)
}
f(5)    # Output: 25 (5 + 20)
print(y) # Output: 10 (global 'y' unchanged)

# Example 2: Lexical scoping in nested functions
make.power <- function(n) {
  pow <- function(x) {
    return(x^n) # 'n' found in enclosing environment
  }
  return(pow)
}
cube <- make.power(3)
cube(2)  # Output: 8 (2^3)
square <- make.power(2)
square(3) # Output: 9 (3^2)

5.2 Optimization Example / Coding Standards

Concept: Writing efficient and readable R code. Good coding standards improve maintainability and collaboration.

  • Prefer vectorization over loops for speed.
  • Pre-allocate vectors if loops are necessary.
  • Use clear, descriptive variable and function names.
  • Use comments and proper indentation. 
# BAD (slow for large n):
sum_sq_loop <- function(n) {
  res <- c() # Dynamic growing
  for (i in 1:n) {
    res <- c(res, i^2)
  }
  return(sum(res))
}

# GOOD (vectorized and efficient):
sum_sq_vectorized <- function(n) {
  return(sum((1:n)^2))
}

# GOOD (pre-allocation if loop is necessary):
sum_sq_preallocate <- function(n) {
  res <- numeric(n) # Pre-allocate vector of n zeros
  for (i in 1:n) {
    res[i] <- i^2
  }
  return(sum(res))
}

5.3 Dates and Times

Concept: R has specialized classes for handling dates and times to perform calculations, formatting, and comparisons correctly.

  • Date class: stores dates without time.
  • POSIXct class: stores date and time as seconds since 1970-01-01 UTC.
  • POSIXlt class: stores date and time as a list of components.
  • Functions: Sys.Date() , Sys.time() , as.Date() , as.POSIXct() , as.POSIXlt() , strptime() , format() .
  • Format codes: %Y , %m , %d , %H , %M , %S , %b , %B , etc. 
# Current date and time
today <- Sys.Date()

Questions & Brief Answers:

  1. Q: What is lexical scoping in R, and how does it determine the value of a variable within a function?
    A: R looks for a variable's value in the environment where the function was defined, not where it was called. It follows a search path from local to parent environments, then global, then loaded packages.
  2. Q: Explain why vectorized operations are generally preferred over for loops for performance in R, using an example.
    A: Vectorized operations are faster because they leverage R's underlying C/Fortran code to perform operations on entire vectors at once, avoiding the overhead of R's loop interpreter. Example: sum(x^2) is faster than a for loop that squares and sums elements one by one.
  3. Q: What R class would you typically use to store a date without time information? How about a date and time?
    A: Date class for dates only. POSIXct or POSIXlt for date and time.
  4. Q: You have a character string "June 02, 2025 10:45 AM". How would you convert it into a POSIXct object in R?
    A: as.POSIXct("June 02, 2025 10:45 AM", format = "%B %d, %Y %I:%M %p") .

Module 6: Loop Functions, Debugging & Simulation

6.1 Loop Functions (Apply Family)

Concept: A set of higher-order functions that provide efficient and idiomatic ways to apply a function over elements of lists, vectors, or margins of arrays/matrices, reducing the need for explicit loops.

  • lapply(X, FUN, ...) : Applies FUN to each element of a list X . Returns a list.
  • sapply(X, FUN, ...) : Simplifies the result of lapply to vector or matrix if possible.
  • apply(X, MARGIN, FUN, ...) : Applies FUN over rows ( MARGIN=1 ) or columns ( MARGIN=2 ) of a matrix or array.
  • mapply(FUN, ...) : Multivariate version of sapply , applies FUN to multiple arguments.
  • tapply(X, INDEX, FUN, ...) : Applies FUN to subsets of vector X defined by factors INDEX .
  • split(x, f) : Splits vector or data frame x into groups defined by factor f . 
my_list_num <- list(a = c(1,2), b = c(3,4,5))

# lapply
lapply(my_list_num, sum)
# Output: list(a = 3, b = 12)

# sapply
sapply(my_list_num, sum)
# Output: a  b
#         3 12

my_matrix <- matrix(1:9, nrow = 3)
# apply (sum of rows)
apply(my_matrix, 1, sum) # 12 15 18

# apply (mean of columns)
apply(my_matrix, 2, mean) # 2 5 8

# mapply
mapply(sum, 1:3, 10:12) # sum(1,10), sum(2,11), sum(3,12)
# Output: 11 13 15

# tapply and split
data_values <- c(10, 12, 15, 11, 20, 18)
categories <- factor(c("A", "B", "A", "B", "A", "B"))
tapply(data_values, categories, mean)
# Output: A: 15, B: 13.666

split_data <- split(data_values, categories)
print(split_data)
# $A
# [1] 10 15 20
# $B
# [1] 12 11 18

6.2 Debugging Tools

Concept: Tools and techniques to identify and fix errors (bugs) in your R code.

  • traceback() : Shows the sequence of function calls that led to an error.
  • debug(function_name) : Puts a function into debug mode to step through code.
  • browser() : Pauses execution at a point for interactive debugging.
  • recover() : Post-mortem debugger to navigate call stack on error.
  • stop() , warning() , message() : Generate errors, warnings, and messages. 
my_buggy_function <- function(x, y) {
  z <- x / y # This will error if y is 0
  print(z)
}

# Example of traceback
# my_buggy_function(10, 0) # This would cause an error
# traceback() # Shows where the error occurred

# Example of debug
debug(my_buggy_function)
# my_buggy_function(10, 2) # Now enters debug mode

# Example of browser()
another_function <- function(data) {
  if (!is.numeric(data)) {
    stop("Data must be numeric.")
  }
  browser() # Pause execution here
  mean_val <- mean(data)
  return(mean_val)

6.3 The str Function

Concept: str() stands for "structure." It's a highly useful function for compactly displaying the internal structure of any R object.

  • Shows class, type, dimensions, names, and a few initial values.
  • Crucial for quickly understanding what an object contains without printing its entire content. 
my_df <- data.frame(
  id = 1:3,
  name = c("A", "B", "C"),
  is_active = c(TRUE, FALSE, TRUE),
  scores = list(c(10, 20), c(30, 40), c(50))
)
str(my_df)
# 'data.frame': 3 obs. of 4 variables:
# $ id       : int 1 2 3
# $ name     : Factor w/ 3 levels "A","B","C": 1 2 3
# $ is_active: logi TRUE FALSE TRUE
# $ scores   :List of 3
# ..$ : num 10 20
# ..$ : num 30 40
# ..$ : num 50

str(c(1, 2, 3)) # num [1:3] 1 2 3
str(list(a = 1, b = "hello")) # List of 2: $ a: num 1, $ b: chr "hello"

6.4 Simulation

Concept: Generating random numbers or data based on specific probability distributions. Crucial for statistical modeling, bootstrapping, Monte Carlo simulations, etc.

  • set.seed(seed_value) : Ensures reproducibility.
  • Random Number Generators: rnorm() , runif() , rpois() , rbinom() .
  • sample() : Randomly samples elements from a vector. 
# Ensure reproducibility
set.seed(42)

# Generate 5 random numbers from a standard normal distribution (mean=0, sd=1)
norm_samples <- rnorm(5)
print(norm_samples)

# Generate 3 random numbers from a uniform distribution between 0 and 10
unif_samples <- runif(3, min = 0, max = 10)
print(unif_samples)

# Simulate rolling a die 10 times
die_rolls <- sample(1:6, 10, replace = TRUE)
print(die_rolls)

# Randomly select 2 names from a list without replacement
names <- c("Alice", "Bob", "Charlie", "David")
selected_names <- sample(names, 2, replace = FALSE)
print(selected_names)

Questions & Brief Answers:

  1. Q: When would you choose lapply() over sapply() ?
    A: When you specifically need the output to be a list or want to avoid sapply 's simplification.
  2. Q: You have a matrix my_data_matrix . How would you calculate the mean of each column using an apply function?
    A: apply(my_data_matrix, 2, mean) .
  3. Q: What is the main purpose of tapply() , and how is it different from apply() ?
    A: tapply() applies a function to subsets of a vector based on categories (factors), performing a "group by" operation. apply() applies a function over rows or columns of an array or matrix.
  4. Q: You've run your code and got an error. What is the first debugging function you should use to get a basic understanding of where the error originated?
    A: traceback() .
  5. Q: You want to generate 10 random integers between 1 and 100, and ensure that if you run the code again, you get the same sequence of "random" numbers. How would you do this?
    A: Use set.seed(some_number) (e.g., set.seed(123) ) before calling sample(1:100, 10) .
  6. Q: Describe the output of str(my_data_frame) and explain its utility.
    A: str() provides a concise summary of the internal structure of my_data_frame , showing its class, dimensions, and for each column, its name, data type, and a few initial values. It's useful for quickly inspecting and understanding the data structure without printing the entire object.
